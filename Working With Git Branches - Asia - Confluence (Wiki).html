      Working With Git Branches

      Skip to end of metadata
      • Created by Bernard Lago on 16-Jul-2015
      Go to start of metadata


      1. Do a git pull of alpha-qa-automation-jspringbot project.

      2. List all branches using: git branch -a
      Administrators-MacBook-Pro-3:alpha-qa-automation-jspringbot alpha$ git branch -a
        desktop-integration-tests
      * master
        remotes/origin/HEAD -> origin/master
        remotes/origin/alpha-qa-automation-jspringbot-1.15.x
        remotes/origin/desktop-integration-tests
        remotes/origin/master
      3. Switch branch using: git checkout <branch_name>
      git checkout desktop-integration-tests
      Administrators-MacBook-Pro-3:alpha-qa-automation-jspringbot alpha$ git checkout desktop-integration-tests
      Switched to branch 'desktop-integration-tests'
      Your branch is up-to-date with 'origin/desktop-integration-tests'.
      4. Once switched, modifications can be done to the scripts.
      i.e. modifying locators for create account (since page elements are different from mobile and desktop)
      5. Commit changes to the branch using: git commit -am ‘<MESSAGE>’
      6. Pull latest changes using: git pull —rebase
      7. Push changes to remote repository using: git push origin <branch_name>
      git push origin desktop-integration-tests
      To switch back to master, do: git checkout master

      Please note that we will be using master for all mobile-related sprint stories and desktop-integration-tests for desktop-related sprint stories.
