      How-To: Manage and Create Crucible Repositories

      Skip to end of metadata
      • Created by Juan Jose del Rio, last modified by Michel Morizot yesterday at 09:42 PM
      Go to start of metadata

      Check-list

      You will need to have:

      • A PWS ticket (Component: Operations) for creating the repository.
      • A product registered in Product WS
      • Crucible Administration rights (Build Engineer or DevOps)

      Summary of actions:

      Step 1. Install the script
      Step 2. Use it

      Install script

      ?
      sudo pip install crucible-automation -i http://artifactory-1.${ENV}.internal/artifactory/api/pypi/public-pypi/simple --trusted-host 'artifactory-1.${ENV}.internal'

       

       

      Script source code

      [http://svn.mit.internal/local/primary/build-management/scripts/crucible]


      Use it

      ?
      $ manage-crucible
        
        Usage:
           manage-crucible.py --(status|add|delete|enable|disable|start|stop) (all|repo_name1[,repo_name2,repo_name3,...])
        
        Modes:
              --add          adds repo from productws. 'all' adds all products from productws.
              --delete       deletes repo from crucible. 'all' deletes all repos. (repo must be stopped and disabled first)
              --enable       enables repo in crucible (repo indexing will auto-start upon crucible startup)
              --disable      disables repo in crucible.  repo must be stopped first (repo indexing will not auto-start upon crucible startup)
              --start        start repo indexing.  repo must be enabled first. (renders a repo undeletable)
              --stop         stop repo indexing  (renders a repo deletable)
              --status       show crucible repo status.  'all' shows all repos.  Otherwise, show substring repo name matches only.

      Example

      manage-crucible.py --add "My Fancy Product" --stop "My Old Fancy Product"
