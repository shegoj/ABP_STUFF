      [RFC] How-To: Integrate Python applications with our SDLC

      Skip to end of metadata
      • Created by Pablo Garcia, last modified by Michel Morizot on 13-Oct-2015
      Go to start of metadata

      This How-To guide describes the standard approach to integrate Python applications in our software development lifecycle.

      • 1Requirements
      • 2Step-by-step guide
      • 3How does it work?
      • 4Examples
      • 5Related articles

      Before going with this guide, it is highly recommended to have a look to How-To: Working with Python repositories. There, we explain how the Python repositories infrastructure works, allowing you to work in the same manner as the CI servers.

      Requirements

      To follow this guide, your product must use the new Parent POM 1.6. Please note that this guide is only intended for Python scripts or in general, products that will be installed later using PyPI.

      Django products are not affected by this, since they don't need to retrieve Python artifacts in Build time, but during deployment.

      Step-by-step guide

      1. Create a standard Maven project
      2. Place your Python code into the folder src/main/python.
      3. Ensure you have a setup.py file under src/main/python. Presence of the setup.py file activates the maven python build profile.
      4. Ensure your POM file contains the property <name>. It will be used to generate the file name.

      The rest is done automatically. 

      How does it work?

      We have included all the required executions in the Parent POM, to ensure all the builds are done in same way. These executions are:

      1. Prepare the Python VirtualENV
      2. Install all the required dependencies
      3. Run tests 
      4. Generate package
      5. Deploy to Artifactory

      Also, we do some additional tasks like changing the "-SNAPSHOT" part from the version field to "-dev", in case of building snapshots. In the Python world, "-SNAPSHOT" is not used.

      Examples

      Bldcreator: Script to generate BLD tickets automatically.
