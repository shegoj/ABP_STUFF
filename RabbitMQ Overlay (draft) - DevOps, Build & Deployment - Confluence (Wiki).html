      RabbitMQ Overlay: What is it?

      rabbitmq-overlay is a collection of scripts used in order to automate installation/configuration of rabbitmq clusters.

      It integrates rabbitmq into our configuration and deployments systems.

      SCM: http://svn.mit.internal/local/primary/build-management/artifacts/rabbitmq-overlay/trunk

      CI: https://ci.corp-apps.com/browse/BUILDMANAGEMENT-RABBITMQOVERLAYPUBLISH

      Content:

      • http://svn.mit.internal/local/primary/build-management/artifacts/rabbitmq-overlay/trunk/src/main/resources/scripts/site.sh

      Main script used for start/stop/install/... It also sets default config values that can be overriden on your cluster.properties file.

      • http://svn.mit.internal/local/primary/build-management/artifacts/rabbitmq-overlay/trunk/src/main/resources/scripts/functions.sh

      Helper functions for rabbitmq management.

      • http://svn.mit.internal/local/primary/build-management/artifacts/rabbitmq-overlay/trunk/src/main/resources/scripts/common.sh

      RABBITMQ-OVERLAY: BigIP configuration

       

      BIGIP Example Configuration
      ?
      virtual POD-broker.@@datacenter@@.internal_http {
         snat automap
         pool POD-broker.@@datacenter@@.internal_15672
         destination <internal_IP>:80
         ip protocol 6
         profiles {
            http_xfrwd {}
         }
      }
       
      pool POD-broker.@@datacenter@@.internal_15672{
         monitor all tcp
         members {
            @@nodes@@:15672{}
         }
      }

       

      HOW-TO: Force a clean redeploy

      In order to force a clean redeploy of a rabbitmq-overlay based product, these steps need to be performed before:

      Clean redeploy
      ?
      $ sudo /etc/init.d/<deploy> stop
       
      $ sudo yum erase rabbitmq-server -y
       
      $ sudo yum install rabbitmq-server -y
       
      Redeploy

       

      HOW-TO: rabbitmq-overlay Integration

       

      Step 1) Install required dependencies/software in the destination server.

       

      Example in Puppet-2
      ?
      $rabbitmqver = "3.1.5"                                                       
      system::package { "rabbitmq-server" : ensure =>"$rabbitmqver-1.el6" }
      system::package { "jq" : ensure =>"1.3-2.el6" }

      Step 2) Create the optional properties files under "config" directory:

      config/@override.name@.properties

      You can also add additional users and assign them specific permissions:

      vhost: Virtual Host which will be created and newly users assigned

      admin: Will enable the user as Administrator

      permissions: Will override user permissions in vhosts. Default is '.*' '.*' '.*'

      Step 3) Add rabbitmq-overlay as a dependency on your pom file:

      pom.xml
      ?
      <plugin>
          <groupId>org.apache.maven.plugins</groupId>
          <artifactId>maven-dependency-plugin</artifactId>
          <version>${maven-dependency-plugin.version}</version>
              <executions>
                  <execution>
                      <id>prepare-deps</id>
                      <phase>validate</phase>
                      <goals>
                          <goal>unpack</goal>
                      </goals>
                      <configuration>
                          <artifactItems>
                              <artifactItem>
                                  <groupId>internal.build</groupId>
                                  <artifactId>rabbitmq-overlay</artifactId>
                                  <version>${rabbitmq-overlay.version}</version>
                                  <type>jar</type>
                                  <overWrite>true</overWrite>
                                  <outputDirectory>${project.build.Directory}/dependencies</outputDirectory>
                              </artifactItem>
                          </artifactItems>
                      </configuration>
                  </execution>
              </executions>
      </plugin>

      Step 4) Add fileSet definitions to your assembly.xml file:

      ASSEMBLY (descriptor.xml)
      ?
      <?xml version="1.0" encoding="UTF-8"?>
      <assembly xmlns="http://maven.apache.org/xsd/assembly-1.1.0-SNAPSHOT.xsd">
          <id>script</id>
       
          <formats>
          <format>tar.gz</format>
          </formats>
       
          <baseDirectory>${server.name}</baseDirectory>
          <includeBaseDirectory>true</includeBaseDirectory>
       
          <fileSets>
              <fileSet>
                  <directory>${project.build.directory}/dependencies/scripts/</directory>
                  <includes>
                      <include>*.sh</include>
                  </includes>
                  <outputDirectory>/bin</outputDirectory>
                  <fileMode>750</fileMode>
                  <directoryMode>755</directoryMode>
                  <filtered>true</filtered>
              </fileSet>
              <fileSet>
                  <directory>${project.build.directory}/dependencies/scripts/</directory>
                  <includes>
                          <include>decrypt</include>
                  </includes>
                  <outputDirectory>/bin</outputDirectory>
                  <fileMode>750</fileMode>
                  <directoryMode>755</directoryMode>
                  <filtered>false</filtered>
                  <lineEnding>keep</lineEnding>
              </fileSet>
              <fileSet>
                  <directory>${project.build.directory}/dependencies/config/</directory>
                  <includes>
                      <include>*.py</include>
                  </includes>
                  <outputDirectory>/config</outputDirectory>
                  <fileMode>755</fileMode>
                  <directoryMode>755</directoryMode>
                  <filtered>true</filtered>
              </fileSet>
              <fileSet>
                  <directory>${project.build.directory}/dependencies/config/</directory>
                  <includes>
                          <include>*.properties</include>
                  </includes>

                  <filtered>true</filtered>
              </fileSet>
          </fileSets>
      </assembly>
